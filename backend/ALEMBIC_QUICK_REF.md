# Alembic Quick Reference

## Essential Commands

```bash
cd backend

# Generate new migration (auto-detect changes)
alembic revision --autogenerate -m "Description"

# Apply all pending migrations
alembic upgrade head

# Revert to previous state
alembic downgrade -1

# Check current revision
alembic current

# View all migrations
alembic history --verbose

# View migration heads
alembic heads
```

## Common Scenarios

### Add a New Field to a Model

```bash
# 1. Modify app/models/models.py
# 2. Generate migration
alembic revision --autogenerate -m "Add email field to User"

# 3. Review alembic/versions/<revision>_*.py

# 4. Apply to database
alembic upgrade head
```

### Fix a Migration (Local Dev Only)

```bash
# Revert last migration
alembic downgrade -1

# Delete the migration file in alembic/versions/

# Re-generate if needed
alembic revision --autogenerate -m "Fixed: Add email field to User"

# Reapply
alembic upgrade head
```

### Deploy to Production

```bash
# Before deploying new code
alembic upgrade head

# With Docker
docker-compose exec backend alembic upgrade head

# With Kubernetes
kubectl exec -it <pod-name> -- alembic upgrade head
```

### Manual Migration (Complex Changes)

```bash
# Create empty migration
alembic revision -m "Complex schema update"

# Edit alembic/versions/<revision>_*.py with custom SQL
# Add logic to upgrade() and downgrade()

# Test locally
alembic upgrade head
alembic downgrade -1
alembic upgrade head

# Deploy
alembic upgrade head
```

## Configuration

**alembic.ini**
- `script_location = alembic`
- Reads `DATABASE_URL` from environment

**alembic/env.py**
- Imports: User, Camera, Incident, Alert models
- Reads: DATABASE_URL from app.config.database
- Supports: Autogenerate from model changes

## Status Checks

```bash
# Current database revision
alembic current

# Pending migrations
alembic heads

# Full history with details
alembic history -v

# Specific revision info
alembic show <revision_id>
```

## Tips & Tricks

- **Always review** autogenerated migrations before applying
- **Test both** `upgrade head` and `downgrade -1` locally
- **Commit migrations** to git alongside model changes
- **Use descriptive** migration messages
- **Never edit** applied migrations in production
- **Document complex** migrations with inline comments

## Troubleshooting Checklist

- [ ] Database connection working? `python -c "from app.config.database import engine; engine.connect()"`
- [ ] Models imported in env.py? Check `alembic/env.py`
- [ ] Migration file created? Check `alembic/versions/`
- [ ] syntax valid? Try `python -m py_compile alembic/versions/<file>`
- [ ] Both upgrade and downgrade work? Test locally first

## Emergency Revert

```bash
# Revert multiple migrations
alembic downgrade -2      # Go back 2 migrations
alembic downgrade base    # Go back to database creation

# Return to head
alembic upgrade head
```
